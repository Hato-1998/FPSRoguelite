## 🔧 백엔드 렌즈 핵심 입장 (Claude)
코어 *메커니즘 골격*(카드 RarityTiers·Scope, Fragment 합성훅, 무기 `FFPSRWeaponStatBlock`, 미션 프레임워크)은 데이터드리븐으로 양산을 잘 받친다. 그러나 "볼륨업"이 **무기/카드 추가에 편향**돼 있고, 진짜 리텐션을 만드는 3축이 코드/설계 골격 없이 비어 있다:
1. **시너지 축** — 같은 축 카드는 코드 0으로 무한 양산되나, 원소/상태이상 등 *새 축*은 적에 상태 저장소·소비 파이프라인이 필요 → **콘텐츠가 아니라 C++ 베이스**. §2-3이 자기-플래그한 "양산 전 축 잠금"의 정체.
2. **적 다양성** — 현재 사실상 단일 `BP_Enemy` + 시간커브(`UEnemyScalingProfile`). 적 정의 DataAsset + 시간 가중 로스터가 선결(콘텐츠 아닌 디렉터 확장).
3. **메타 스파인** — 메타/보스/맵은 양산 콘텐츠가 *담길 그릇*. 그릇(SaveManager·맵정의·보스정의) 없이 양산하면 게임에 안 붙는다.
→ 재미 게이트를 **양산 강제 관문**으로. 게이트 전 양산은 프리즈/루프 재설계 시 카드 콘텐츠 전부를 흔든다.

## 🎮 클라이언트 렌즈 핵심 입장 (Codex)
"방향은 맞지만 **콘텐츠 양산 시작엔 아직 이르다**." 가장 위험한 착각 = *무기/카드가 많으면 리텐션이 생긴다*. 이 장르 반복동기 = `빌드 축 × 적 조합 × 보스/맵/메타 목표`. 지금은 빌드 *메커니즘*은 있는데 빌드 *축*이 얕고, 적 조합은 거의 없고, 보스/메타/맵은 그릇 단계.
- 시너지: 수치+Fragment로 *1차 재미*는 나오나 장기엔 납작 → **얇고 선명한 상태이상 3~4개**(Burn/Shock/Frost/Rupture). 단 적마다 Niagara 풀부착 금지.
- 적 다양성이 양산 1순위(무기 8종이면 게이트 표본 충분). 최소 로스터: Grunt/Runner/Tank/Spitter/Exploder/Shield/Elite + **시간대별 가중풀**.
- 프리즈 페이싱 = 게이트 불합격 가능성 **높게** 봄 → 하이브리드 강하게 열어둠.

## 토론 로그 요약
- **R1**: 4쟁점(시너지축·적다양성·메타스파인·게이트선행) 제시. Codex가 전 쟁점 동의 + 구체화(적 7종 로스터, 상태이상 4종 + 경량 피드백 규칙, 프리즈 페이싱 임계치, "완전정지 아닌 게이트용 프로토타입 콘텐츠 허용").
- **R2**: 백엔드가 남은 2긴장 제기 — (A) **게이트와 시너지축 선후 모순**(축 없이 게이트 돌리면 빌드체감 거짓판정) → **게이트 2분할** 제안. (B) **경량 상태이상이 적500 예산/풀링과 충돌** → 상태는 "콘텐츠 2~3개"가 아니라 **경량 상태 서브시스템(배치틱/풀리셋/연쇄캡)** C++ 베이스. Codex 양쪽 동의 + **순서 보정**: 프리즈 하이브리드 최소형은 **G1 판정 *전*** 에 들어가야 페이싱 판정이 공정(아니면 이미 아는 리스크 재확인에 그침). → **수렴**.

## ✅ 합의 권고
1. **재미 게이트 = 양산 강제 관문 + 2분할**:
   - **G1**(지금 가능): 스웜 사격 손맛 · 프리즈/하이브리드 페이싱 · 1인칭 사각 위협 · 적 300~500 성능. = "계속 쏠 만한가" → **양산 가부 결정**.
   - **G2**(상태축 프로토 후): 상태이상 2~3개 + 카드 선택으로 플레이 양상이 달라지나 · Fragment×상태축 물림. = "반복 플레이할 빌드 이유가 생겼나".
   - `Roadmap.md` §7-5 단일 게이트를 G1/G2로 분할 기재.
2. **양산 1순위 = 적 다양성** (무기/카드 추가 아님). 데이터드리븐 **EnemyDefinition + 시간 가중 로스터 디렉터** 선결. 최소 로스터 Grunt/Runner/Tank/Spitter/Exploder/Shield/Elite(엘리트=소수라 GAS/StateTree 허용).
3. **상태이상 = 경량 상태 서브시스템 C++ 베이스**(콘텐츠 아님): 적엔 작은 `ActiveStatuses` struct만, **만료/DoT/스택은 중앙 서브시스템 배치틱**(틱 대상=상태 보유 적만), 풀 반환/재활성 시 멱등 클리어(상태·틴트·pending chain·DoT 누산), 연쇄/폭발은 **서버권위 + 이벤트당 타겟수/반경/프레임/동시발생 캡**, Significance 저티어는 VFX만 생략(데미지·만료는 데이터상 진행).
   - **가시 피드백 하한선**(1인칭 수백): 색 틴트만으론 부족(화면 노이즈) → 공통=짧은 머티리얼 틴트/파라미터 + 피격숫자/히트마커 변주. Burn=주황틴트+사망폭발/전이만 실VFX+묶음사운드. Shock=전이 VFX는 S0/S1만, S2/S3는 사운드/히트마커. Frost=청색틴트보다 **느려진 실루엣/애님속도** 우선. Rupture=누적표식 약하게+터질 때 큰 피드백. **머리위 인디케이터는 S0/S1 근접 위협 적에만** 작은 아이콘/링 허용, 나머지는 틴트/히트마커/사운드 집계.
4. **프리즈 하이브리드 최소 프로토타입을 G1 *전* 에**: 일반 수치카드=Q/E(또는 1/2/3) 비동기 전투지속 / 무기·Fragment·희귀변형·미션보상=프리즈 유지 / 다중레벨업=순차·보류 / 보스전 일반레벨업 프리즈 금지는 후보 플래그. G1에서 전역프리즈 baseline vs 하이브리드 A/B 동시 판정.
5. **맵**: 실제 맵 양산은 후순위. **맵 선택/정의 데이터 골격만 메타 스파인과 함께 선행**.
6. **확정 골격 순서(양산 직전)**:
   `① G1 정의/계측 정리(통과·실패 기준·FPS/적수/프리즈빈도/선택소요시간 메모 고정) → ② EnemyDefinition + 시간가중 로스터 디렉터(데이터구조 먼저, 프로토 3종) → ③ 프리즈 하이브리드 최소 프로토 → ④ G1 게이트(양산 가부 결정) → ⑤ 경량 상태 서브시스템(배치틱/풀리셋/연쇄캡/Significance 피드백, 상태 2~3개 프로토) → ⑥ G2 게이트 → ⑦ 메타 SaveManager + 맵정의 + 보스정의 골격`
