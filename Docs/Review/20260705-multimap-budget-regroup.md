## 🎮 클라이언트 렌즈 핵심 입장 (Codex)
- **명제(수렴 축): "다중맵=자유를 주되, 본게임 압력·보상·burst는 '2명 이상 같은 전선'에서 선명하게 켜진다. 솔로 분산=정찰 모드. USP는 2인+ 같은 전선에서 완전히 산다."**
- 예산 회수는 **Kill 아닌 조용한 디스폰**(안 보이면 존재하지 않은 사건). 진입 시드는 "즉시 적"이 아니라 **0~3초 내 측후방/문밖 사운드로 압력**(눈앞 팝업 금지).
- rally=강제 아닌 **명시 rally pad**(후회 비용 낮추기). 인센티브는 "흩어지면 몰래 빼앗김"(숨은 페널티=금지)이 아니라 "뭉치면 편/선명/빠름"(양성 당김)만.

## 토론 로그 요약 (라운드별 핵심)
- **R1**: 3문제 프레이밍. Codex가 Kill→silent recycle, 자동텔포→명시 rally pad, "진입시드=0~3초 압력(즉시 아님)", 인센티브 양성/음성 경계 확립.
- **R2**: 2단 예산 구조(예약 보증 + 백그라운드 recycle 리필 + 점유맵 밀도 플로어) 합의. rally=소수전용·짧은페이드·피격취소. 자석=근접 smoothing(버프 아님).
- **R3**: 스트레스 3케이스. **완전분산(1/1/1/1)=수용하되 지원 안 함**(솔로맵 플로어만·낮게). 전환 경계=a+b 하이브리드(추적자 소수). recycle=플로어+교전보호+drain rate(≤10-15%/10s)+부족시 burst 축소(1:1 즉시대체 아님).
- **R4**: 티어링. 첫 세션=Tier 0+최소 진입시드. "빈 맵 하드드레인·새 점유맵 nonzero seed"=Tier 0 승격. "최대그룹크기 예산배분"=Tier 1(allocator hook만 Tier 0).
- **R5**: **allocator v0 정책도 Tier 0 승격**(임시 "2인+ 맵>솔로 맵" 가중치 없으면 명제 검증 불가). Tier 0 생존조건 3개 추가. 시작 수치 미세조정. (c) 디렉터 결정 5항목 분리.
- **R6(디렉터 결정 반영·검증)**: §🙋 6항목 결정 확정(아래 ✅). **핵심 = #5로 그룹 버프 전면 폐기 → "뭉치면 효율"(#2)은 오직 allocator가 고가치 콘텐츠(미션/보스/엘리트/연쇄)를 2인+ 그룹에 집중시키는 것으로만 성립**(버프 아닌 "콘텐츠 접근 효율"). allocator 애매 시 1/1/1/1 저층 파밍이 수학적 최적→붕괴 = **이 설계의 진짜 심장은 텔포가 아니라 allocator.** 텔포=쿨타임만으론 남용 미방지(+장치·앵커·채널·shared 쿨타임 필요). 맵 잔존=풀액터 유지 아님(dormant/HISM 경량화 필수).

## ✅ 합의 권고 (양 렌즈 동의)
**설계 명제**: 다중맵=자유 / 본게임(압력·보상·burst)=2인+ 같은 전선에서 선명 / 솔로=정찰 모드 / 하드 테더 없음. (3줄 근거: ①제1원리=적 200 전역예산 안에서 "플레이어 있는 전선"만 싸게 산다 ②UE표준=월드 스트리밍은 따르되 적 예산은 Lyra per-actor 아닌 map-aware swarm allocator로 이탈 ③정합="2인+=본게임·솔로=정찰"을 예산 배분 규칙으로 먼저 박아야 후속 rally/보상이 의미.)

**구현 티어(첫 세션 스코프 확정)**
- **Tier 0 (첫 세션, 코어+생존조건)**: 레벨 스트리밍(문파괴→인접맵 스트림-in, persistent world 다른 좌표) · per-map flow-field 등록/evict · 전역 캡 + **map-aware allocator v0**(임시 "2인+ 맵 > 솔로 맵" 가중치 포함) · NetCull 구현 · 미션/스폰 점유맵 파라미터화 · **최소 진입 시드**(새 점유맵 nonzero, 수초 내 스폰포인트 출현) · **빈 맵 target=0 + 빈 맵 적 하드 드레인** · **문 파괴 상태 서버권위 복제/late-join** · **전환 중 점유맵 판정 grace(몇 초 old/new 이중점유)** · **stream-in 실패/지연 fallback(낙하·빈 전장 방지)**.
- **Tier 1 (바로 다음, 예산 게임필)**: 동적 burst reserve(평시 10-15·진입 5-10s 20-30) · silent recycle + 안전게이트(NetCull밖·LOS없음·최근교전 10-15s·미션/엘리트 보호) + drain rate(첫값 10%/10s) + **local pressure floor**(count 아닌 압력·후방 보존) · 점유맵 밀도 플로어(솔로 20-30)/상한 · 전환 추적자(a+b, 추격토큰 재스폰, 솔로 ≤4/2인 6-8/3-4인 8-12·cap 12, 빠른 잡몹만·엘리트 금지) · **최대그룹크기 예산배분 정책 확정**.
- **Tier 2 (콘텐츠/밸런스)**: rally pad(소수전용·파괴문·채널 0.75s·페이드 0.45-0.6s·강타취소) · split 감지(1/1/1/1·15-25s)→rally 강조·다운HUD·솔로 억제(느린/시끄러운 솔로 문돌파) · 양성 인센티브(자석 smoothing·큐 공유, 버프 아님·UI문구 금지·전역흡입 금지) · 보스=rally 이벤트 · 문리셋 debt.

**시작 수치(첫 플레이테스트 출발값, 전부 튜닝 대상)**: 캡 200 / reserve 10-15(진입 burst 20-30) / 솔로 플로어 20-30 / 추적자 cap 12(솔로 ≤4) / drain 10%/10s / rally 채널 0.75s / 페이드 0.45-0.6s / split 감지 15-25s.

## ⚖️ 미해결 쟁점 (튜닝으로 수렴, trade-off 잔존)
- **예약 % vs 기존 맵 압력**: burst reserve가 크면 기존 맵 압력이 얇아짐 → "동적 burst(상시 유휴 아님)"로 완화하나 정확 균형은 실측 튜닝.
- **recycle 체감**: 플로어+교전보호+drain rate로 "압력 스르륵 얇아짐"을 억제하나 완전 제거는 실측 필요(1:1 즉시대체는 고무줄이라 예외 장치로만).

3. **rally pad 정체성**: 구제 장치 vs 전략적 재집결 버튼 vs 페널티 회피권.
4. **분산 판타지 태도**: 각자 방 찢는 자유를 멋지게 vs 협동 긴장 훼손으로 억제.

## ✅ 디렉터 결정 확정 (2026-07-05, R6 반영)
§🙋 6항목 → 디렉터 결정:
1. **솔로 분산 = 정찰로 품기(약한 억제)** — per-map 미션이 이미 억제(다인 미션은 분산 시 불가)라 추가 명시 억제 없음. "은근한 비효율"만 확장(아래).
2. **2인+ 보상 = "효율 좋음"**(무조건 뭉침 강제 반대). ※ **"2인+" = 파티 크기 아닌 "뭉침 임계값"**(2·3·4 전부 포함), 게임 = 1~4인. allocator는 그룹 크기 비례.
3. **뭉침 어드밴티지 = 전면 반대** — 근접 버프·자석 smoothing·큐 공유 **전부 폐기**. 뭉침 유인은 오직 **콘텐츠 배분(구조)**으로만.
4. **텔레포터 = 장치 활성화 → 인터랙트 → 대상 플레이어 지정 → 텔레포트, 쿨타임.** R6 보강(쿨타임만으론 남용 미방지): 도착 앵커(대상 근처 안전지점) · 짧은 채널(0.75~1.5s, 피격/넉백/DBNO 취소) · 전투중 즉시탈출 제한 · **개인 outbound + 장치/팀 shared 쿨타임** 중 최소 하나 · 대상 카드 UI(이름/맵/미션/위험/도착가능). 기본 선택=최대 그룹 or 활성 미션 보유자.
5. **문 = 혼자선 부수기 어려운 체력**(같이 부수고 이동 전제; 솔로 앞서가도 텔포로 재집결) — 좋은 soft group-gate(채택).
6. **맵 경계 = 언로드 안 함 · LOD 컬 · 잔존.** XP/픽업 LOD-컬 잔존(이관/소멸 로직 불요), **적만** 빈 맵 드레인(적=비쌈: 시뮬+복제). ⚠️ 잔존 ≠ 풀액터 유지 → dormant/HISM/경량 데이터로 접기 필수. 파일럿 메모리 게이트. 보너스=텔포 목적지 맵 항상 로드→크로스맵 이동 공짜.
7. **분산 판타지 태도 = 하이브리드 (명시 확정 2026-07-05)**: **짧은 정찰 분산 = 자유(멋짐, 다이브 픽션과 정합) / 장기 완전분산 = 은근히 비효율(정찰 모드로 수렴)**. 명시 페널티·그룹 버프 없이 "게임은 그룹 있는 곳에서 일어난다"로만 되돌린다. → 이게 #1~#6의 상위 스탠스이며, 위 결정들과 완전 정합.

**⚠️ R6 핵심 = allocator가 이 설계의 심장**: #5로 그룹 버프 전면 폐기 → **"뭉치면 효율"(#2)은 고가치 콘텐츠(미션/보스/엘리트/연쇄)가 2인+ 그룹에 자연 집중되는 것으로만 성립.** ∴ **Tier 1 "최대 그룹 크기 배분"은 적 예산뿐 아니라 미션/이벤트 배치까지 포함**(콘텐츠-aware allocator). 규칙: "솔로도 손해 안 보지만, 본게임급 사건은 2인+ 있는 곳에서 더 자주·조밀·연쇄."

**은근한 비효율 채택 리스트**(명시 페널티 아님·그룹 버프 아님 — "게임은 그룹 있는 곳에서 일어난다"): 최대 그룹 우선 미션 큐 · 솔로맵=정찰 태그(문/루트/보스 단서 선발견, 즉시 고수익 회수는 어려움) · 다역할 오브젝트(채널+방어+문파괴) · 연쇄 미션 거리(성공 미션 근처 다음 미션) · 문/장애물 시간세(솔로 가능하나 탄창/시간/위험) · 보스 추적선(보스가 다수 전장으로 끌림) · 스카우트 보상=정보 중심(루트 단축, XP 폭발 아님). **⛔ 제외(디렉터 2026-07-05, 게이트/페널티 인접)**: ~~전투 열기/소음 누적~~(솔로 보상 감소로 읽힘) · ~~고가치 이벤트 최소 인원 조건~~(솔로 진입 차단=하드 게이트). **금지선(숨은 페널티=배제)**: 솔로 XP 배율↓ · 적 체력/드롭 몰래↓ · 솔로 미션 타이머 불리 · 솔로 텔포 쿨타임↑ · 솔로맵 스폰 굶김 · "혼자면 약화" 상태효과.
