# 반동 P3 패턴 저작 · PIE 스모크 — 사용자 에디터 가이드

> 브랜치 `phase/recoil-crystalrecoil`. P2(확산 단일소스+서버 heat 패리티) 코드 + P3 heat 곡선(원거리 무기 6종)은 **헤드리스로 완료·커밋**됨(`19ef50c`). 남은 건 **반동 패턴 저작(에디터)** 과 **PIE 스모크**뿐이고, 이 둘은 사용자 작업입니다. 끝나면 알려주시면 제가 P4(정리·검증·머지)를 마무리합니다.

---

## 지금 상태 (자동 완료분)

- **확산(Bloom)**: 이제 무기별 heat 곡선이 구동합니다. Rifle/SMG/LMG/Sniper는 연사 시 확산이 벌어지고(기존 값 그대로 이관), Shotgun/Bazooka는 고정 확산(원래 블룸 없음). **확산은 지금 바로 PIE에서 확인 가능**합니다.
- **반동(카메라 킥)**: 아직 **패턴 미저작이라 일반 무기는 반동이 없습니다**. 아래 "할 일 A"에서 저작하면 살아납니다. (ChargeLaser는 자체 차징 램프라 반동 있음, Knife는 무반동)

## 할 일 A — 무기별 반동 패턴 저작 (CrystalRecoil 비주얼 에디터)

### A-1. 패턴 에셋 만들기 (무기당 1개)
1. 콘텐츠 브라우저에서 `Content/Weapons/` 아래 적당한 폴더(예: `Recoil/`) 우클릭 → **Add → Gameplay → Recoil Pattern**.
2. 이름: `RP_Rifle`, `RP_SMG`, `RP_LMG`, `RP_Sniper`, `RP_Shotgun`, `RP_Bazooka` 권장.
3. 더블클릭 → **Recoil Pattern Editor**가 열립니다.

### A-2. 패턴 에디터에서 스프레이 그리기
- 원점(중앙)이 **첫 발이 떨어지는 곳**. **Shift+클릭**으로 유닛(발)을 **누적 위치**에 찍습니다. 각 유닛의 `Position`은 "그 발까지 크로스헤어가 이동한 **누적 좌표**"(X=우측 Yaw°, Y=위 Pitch°) — 벽에 찍히는 탄착 궤적을 그리는 겁니다.
  - ⚠️ 플러그인 헤더 주석엔 "delta from previous unit"이라 적혀 있지만, **실제 런타임 `ConsumeShot`은 연속 유닛의 *차이*를 발당 킥으로 씁니다** → 저장값은 **누적**입니다(v2.0 오버홀 후 주석이 stale). 예: 유닛이 (0,0)→(0,2)→(0.3,3.6)이면 발당 킥은 [위2°] 다음 [위1.6°+우0.3°].
  - 그래서 **눈으로 궤적을 그리면 됩니다**(위로 오르다 좌우 지그재그). 숫자를 직접 넣을 땐 **델타가 아니라 누적 좌표**로.
- 단축키: **Shift+클릭**=유닛추가, **R**=자동정렬, **S**=스케일, **F**=화면맞춤, **Shift+S**=스냅토글, **H**=단축키표시.

### A-3. ⚠️ 세기(magnitude) 규약 — 중요
패턴의 Y값(도)은 **"조준(ADS) 풀세기 기준의 실제 상승 도수"** 로 그리세요. 런타임에 이렇게 곱해집니다:

```
실제 킥 = 패턴Position × (힙이면 HipVerticalScale, 조준이면 ADSVerticalScale) × 반동카드스케일
```

- 즉 **힙파이어는 패턴이 자동으로 약해집니다**(HipVerticalScale 기본 0.4 → 힙은 40% 세기). 그러니 패턴은 ADS 기준으로 시원하게 그리세요.
- **`RecoilVertical`(무기 스탯)은 이제 "숫자 자체"가 아니라 "세기 레버"** 입니다. 반동↓ 카드가 `RecoilVertical`을 낮추면 → 패턴 전체가 비례 축소(카드스케일<1). 패턴 shape는 그대로, 세기만 줄어듦. (그래서 반동↓ 카드가 여전히 작동합니다 — §2-4-2 캐주얼화 레버)
- 따라서 **패턴 저작 시 `RecoilVertical` 기본값은 안 건드려도 됩니다**(현재 값이 카드스케일의 기준점). 세기 자체를 바꾸려면 패턴 Y값 or Hip/ADSVerticalScale로.

### A-4. 무기별 권장 시작값 (궤적을 그리고 PIE로 다듬으세요)
> 값은 **조준(ADS) 풀세기 기준 누적**. 힙파이어는 자동 ×0.4(HipVerticalScale). "총 상승"=마지막 유닛의 누적 Y(°), "좌우 폭"=지그재그 진폭(°, 우→좌 번갈아).

| 무기 | 유닛수 | 총 상승(°) | 좌우 폭(°) | PatternEndBehavior | UpliftSpeed / RecoveryDelay | 성격 |
|---|---|---|---|---|---|---|
| **Rifle** (연사 FR8) | ~8 | ~8 | ±0.8 | **RestartFromCustomIndex** (Idx≈4) | 0.7 / 0.12 | 초반 수직→후반 좌우, 학습가능 |
| **SMG** (연사 FR16) | ~6 | ~5 | ±1.0 | **Random** (X±0.8, Y0.4~0.8) | 0.75 / 0.1 | 빠르고 산만 |
| **LMG** (연사 FR15) | ~6 | ~10 | ±1.2 | **Random** (X±1.0, Y0.6~1.2) | 0.6 / 0.15 | 강한 초반→후반 카오스 |
| **Sniper** (단발 FR1.2) | 1~2 | ~5 | ~0 | Stop | 0.9 / 0.1 | 단발 강킥 |
| **Shotgun** (단발 FR1.5) | 1~2 | ~4.5 | ~0.3 | Stop | 0.85 / 0.1 | 강한 한 방 |
| **Bazooka** (AOE FR8) | 1~2 | ~3 | ~0.3 | Stop | 0.8 / 0.12 | 중강 킥 |

**⚠️ 탄창이 커지는 우리 게임에서 특히 중요** — 패턴 유닛 수는 **탄창 크기와 무관**합니다:
- 유닛은 **초반 특징 스프레이**만 그림. 그걸 넘는 긴 연사(업그레이드로 탄창 40+)는 `PatternEndBehavior`가 무한정 이어받습니다. `ConsumeShot`: ShotIndex가 유닛 수를 넘으면 Stop(정지)/RepeatLast(마지막 델타 무한반복)/RestartFromCustomIndex(지정 인덱스로 루프)/Random(절차적 스프레이).
- **RepeatLast는 지양** — 큰 탄창에선 화면 밖으로 무한 상승. 연사총은 **RestartFromCustomIndex**(지속사격 구간 루프) 또는 **Random**(절차적 스프레이).
- **업그레이드로 탄창이 커질수록 탄 대부분이 이 end-behavior 구간에 떨어지므로, 후반 루프/랜덤 튜닝이 late-game 손맛을 좌우합니다.** (초반 패턴은 "첫 몇 발 정체성")
- 확산(heat)도 별개로 `MaxRecoilHeat`에서 상한 → 탄창 커져도 최대치까지만 벌어지고 유지.
- 패턴은 **트리거를 새로 누를 때** + **재장전이 시작될 때** ShotIndex 0으로 리셋됩니다(✅ 구현됨). 즉 홀드 중 자동/수동 재장전을 거치면 새 탄창은 **초반 패턴부터 다시** 뿌립니다(end-behavior 지속 아님). 오너-로컬·전 무기 공통(근접/차지레이저 제외). 확산 heat는 재장전 무사격 구간에 알아서 식으므로 별도 리셋 불요.

기타 회복(InitialRecoverySpeed 2 / MaxRecoverySpeed 10 / RecoveryAcceleration 40 / RecoveryCancelThreshold 0)은 기본값 시작.

### A-5. 무기 DA에 패턴 배선
각 무기 DA(`Content/Weapons/DataTable/DA_Weapon_<무기>`)를 열고 **`무기|반동 → 반동 패턴(CrystalRecoil)`**(`RecoilPattern`) 필드에 방금 만든 `RP_<무기>`를 지정 → 저장.
- **ChargeLaser·Knife 는 비워 두세요**(패턴 없음이 정상).

## 할 일 B — PIE 스모크 체크리스트

각 무기를 들고 PIE에서 확인:

**확산(Bloom) — 지금 바로 확인 가능**
- [ ] Rifle/SMG/LMG 연사 시 크로스헤어가 **점점 벌어짐** → 손 떼면 잠시(0.15s) 뒤 **다시 좁아짐**.
- [ ] **조준(우클릭)** 시 확산이 **좁아짐**(ADSSpreadMultiplier).
- [ ] Shotgun/Bazooka는 확산이 **안 벌어짐**(고정) — 정상.
- [ ] **확산↓/`SpreadDegrees` 카드** 먹으면 기본 확산이 줄어듦(레버 유지 확인).

**반동(카메라 킥) — 패턴 저작 후 확인**
- [ ] 연사 시 화면이 패턴대로 상승/드리프트. 마우스로 **눌러 보정(compensation)** 되고, 손 떼면 **회복**.
- [ ] 힙파이어가 조준보다 약함(Hip/ADSVerticalScale). **반동↓ 카드** 먹으면 킥이 작아짐.
- [ ] ChargeLaser는 차징 중 서서히 상승(기존 유지). Knife 무반동.

**게이팅**
- [ ] 카드 선택 **프리즈** 중 발사/반동/확산 정지. **DBNO(다운)** 중에도 정지.

**서버 패리티 (여력 되면, 리슨서버 2인+)**
- [ ] 비호스트(원격) 플레이어가 연사할 때 **실제 탄착이 화면 크로스헤어 확산과 일치**(서버가 더 좁게 쏘지 않음). 이게 P2 핵심 수정 지점입니다.

## 참고 — heat/확산 시스템 튜닝법 (DA `무기|반동` 필드)

이미 저작돼 있지만, 나중에 무기 확산 느낌을 바꾸려면:
- **`확산 heat: 발당 heat증가`**(ShotToHeatCurve): 발당 확산 증가량(도). 상수 곡선이면 매 발 고정 증가. X축=현재 heat라, "많이 벌어졌을 땐 덜 증가"를 원하면 우하향 곡선.
- **`확산 heat: heat→확산각`**(HeatToSpreadAngleCurve): heat(=우리 규약상 도)를 실제 확산각(도)으로. **heat=0 → 0° 앵커 필수**(안 그러면 발사 전에도 확산 넓어짐 — IsDataValid가 경고).
- **`확산 heat: heat→냉각`**(HeatToCooldownPerSecondCurve): 초당 식는 heat(도). 클수록 빨리 좁아짐. **음수 금지**(폭주 — IsDataValid 에러).
- **`최대 heat`**(MaxRecoilHeat): 최대 동적 확산(도).
- **`냉각 지연`**(RecoilHeatCooldownDelay): 마지막 발사 후 이 시간(초) 뒤 냉각 시작. 크면 연사 중 확산이 더 오래 유지.
- **규약**: 현재 전 무기 `heat = 확산 도(°)` 로 저작(1:1). 3곡선은 **전부 채우거나 전부 비우거나**(부분 저작 시 IsDataValid 에러).
- 헤드리스로 다시 저작하려면 `Scripts/recoil_p3_heat_curves.py` 참고(값 바꿔 재실행).
