## §0 검증 결과 — 이 문서의 숫자를 믿어도 되는가

리포트가 근거로 쓴 값들을 **소스에서 직접 대조**했다.

| 주장 | 판정 | 실제 |
|---|---|---|
| 플로우필드 셀 200cm | ✅ 확인 | `FPSRFlowFieldComputer.h:41` `CellSize = 200.0f` |
| 적이 넘는 한계 45cm | ✅ 확인 | `FPSRFlowFieldComputer.h:271` `DefaultClimbableStepHeight = 45.0f` |
| 적 이동 속도 250cm/s | ✅ 확인 | `FPSREnemyBase.h:245` `MoveSpeed = 250.0f` |
| 스폰 시야(FOV) 게이트가 제거됨 | ✅ 확인 | `FPSREnemySpawnSubsystem.cpp:782·1196·1219` 주석이 "FOV gate was REMOVED (2026-06-29)" 명시 → **적이 눈앞에 튀어나오지 않게 하는 책임은 100% 배치에 있다** |
| `bake_building()`이 건물 1채당 홀더 1개 | ✅ 확인 | `fpsr_citygen.py:446~475` |
| 지상층 분리가 `'_base_' in name` | ✅ 확인 | `fpsr_citygen.py:231`. `SM_Bld_Wall_Shop_*`는 이 조건에 안 걸려 **상층으로 간다**(상가 벽을 풀에 넣을 때 손봐야) |
| 셀당 다층(NumLayers) 존재 | ✅ 확인 | `FPSRFlowFieldComputer.cpp` 전반 |
| **적 동시 200~300, 캡 500** | ❌ **틀림** | `FPSREnemySpawnSubsystem.h:312` **`GlobalAliveCap = 200`**. SeedReserve를 빼면 **실질 192**. 300도 500도 불가 |
| **원거리 적 밴드 39m** | ❌ **틀림** | 근거 없음. 실제 = 근접 `AttackRange=150`(1.5m) · 원거리 `RangedEngageRange=1400`(**14m**). 264m 직선은 14m의 **18.9배**라 문제는 오히려 더 크다 |
| 메시 이름 26개 표본 | ✅ 25개 실존 | 유일한 오류 = `SM_Bld_Wall_Shutter_Closed_01` → 실제는 `SM_Bld_Wall_Shop_02_Shutter_Closed/_Open` (**Shop_02용 1쌍뿐**) |

**아래 본문은 위 오류 3건을 고쳐서 실은 것이다.** 그 밖의 수치 중 "미실측"이라고 적힌 것
(벽 모듈 삼각형 수, 간판 크기, 전선 길이, 계단 단높이)은 **아직 안 잰 값**이니 그대로 믿지 말 것.

## §1 한 줄 요약 — 지금 가장 부족한 것 3가지

| # | 부족한 것 | 지금 상태 | 이게 없으면 생기는 일 |
|---|---|---|---|
| **1** | **Bake가 도시 규모를 못 버틴다** | `fpsr_citygen.bake_building()`이 **건물 1채당 홀더 1개** (446행) | 건물 64~380채 → 드로우콜 400~2,000. **예산 920을 건물만으로 다 먹는다.** 5개 조사축이 전부 이걸 1순위로 지목 |
| **2** | **높이가 없다 = 도시가 아니라 평지다** | 건물 0채 | 골목 5m / 큰길 20m인데 벽이 없어 협곡비(H/W)가 0. 시야 차단이 전혀 없어 **264m 직선이 그대로 뚫린다**(원거리 적 교전 사거리 **14m**의 **18.9배** — 리포트 원문의 '39m'은 근거 없음, 소스 실제값 `RangedEngageRange=1400`) → 스웜이 다 보여 압박 소멸 + 200마리가 한 화면에 들어와 GPU 최악) |
| **3** | **게임플레이 골격이 하나도 없다** | 스폰 포인트 0 · 보스 아레나 0 · 병목 0 · 커버 0 | 소블록 한 바퀴 = 220~240m = 적 속도 250cm/s로 **96초 무한 카이팅**. 200마리가 설 면적(100~130㎡)도 미확보 |

## §3 순서와 이유 — 이 순서를 어기면 재작업

```
S0 (툴 수정)  ─┐
S1-R (도로)   ─┴─→ S2 (블록 규격) ─→ S3 (건물) ─→ S4 (게임플레이) ─→ S5 (디테일) ─→ S6 (조명) ─→ S7 (Bake)
```

| 순서 | 왜 먼저인가 | 어기면 |
|---|---|---|
| **S0-1(Bake 병합) → 모든 건물 작업** | 채당 홀더 구조로 64채만 지어도 드로우콜 400~500, 380채면 2,000+ | 건물 다 짓고 나서 **전량 재생성** |
| **S1-R1(도로 3단) → S2** | 도로 폭이 바뀌면 블록 윤곽·건물 위치·간판 돌출 비율이 전부 바뀐다 | 건물 한 채라도 놓으면 **S2~S6 전부 재작업** |
| **S1-R2(폭 하한) → S2** | 블록 윤곽 설계의 입력값 | 골목이 좁아 적이 안 들어옴 → 블록 재배치 |
| **S4-4(소품 실측) → S4-6·S5** | 45~60 함정 화이트리스트 없이 배치하면 어디가 함정인지 사후 특정 불가 | 소품 전량 재검사 |

| **S5-6(간판 태그 규약) → 사용자 배치** | 태그 없이 놓은 100개는 S7 Bake가 흡수 못 한다 | 드로우콜 +100, 수작업 재태깅 |

## §6 지금 결정해야 할 것 5가지

### 결정 1 — 도로 3단 위계, 지금 할까? (**가장 급함**)

|---|---|---|

| B | 안 함, 2단 유지 | 나중에 하려면 **S2~S6 전부 재작업**. 4인 협동 길 찾기가 "골목이 다 똑같다"가 됨 |

### 결정 2 — 육교/2층에 콜리전을 줄까?
| 선택 | 내용 | 결과 |
|---|---|---|
| **A** | **BlockAll** — 적이 계단 타고 올라옴 | 도피처 봉쇄 = 서바이벌 압박 유지. **단, 계단 단높이 실측이 선행 조건**(60cm 넘으면 적은 못 올라가고 플레이어만 = 무적지대) + 플로우필드 셀 2배 |
| **B (안전 기본값)** | **NoCollision** — 순수 장식, 플레이어도 못 올라감 | 스웜 영향 0, 실측 불필요. 2층 동선은 랜드마크 1~2곳만 나중에 시험 |
| C | 육교 자체를 안 씀 | S4-2 시야 차단을 차량/잔해로만 |

### 결정 3 — 간판 물량, 사용자가 다 놓을 수 있나?
레퍼런스 밀도 적용 시 **대형 240 + 소형 1,400개**다.
| 선택 | 내용 |
|---|---|
| **A (권장)** | **절충** — 대로변 정면만 사용자 직접, 나머지는 규약 기반 자동 배치 |
| B | **빈 간판 판만 자동 배치** → 사용자가 골라서 교체 |
| C | 전량 사용자 직접, 밀도 목표를 **1/5로 하향**(대형 50 + 소형 300) |
| — | 어느 쪽이든 **배치 시점 태그 `CityLayout:S6` 규약을 먼저 전달**해야 S7 Bake가 흡수한다 |

### 결정 4 — 블록 내부를 어떻게 채울까?
| 선택 | 내용 | 대가 |
|---|---|---|
| **A (권장)** | **링 셸** — 도로변 한 겹만 모듈, 안쪽은 블록아웃 매스 + 지붕 캡(NoCollision) | 조각 1/3~1/4, 플로우필드 유효 셀 감소. 옥상 동선 사라짐, 블록아웃 1500 단위 스케일 보정 필요 |
| B | 블록 전체를 모듈로 채움 | 조각 3~4배 = 예산 위험 |
| C | 링 + **안쪽을 중정으로 개방** | KWC 채광정 느낌은 나지만 **스웜이 갇히는 주머니** 발생 → 권장 안 함 |
