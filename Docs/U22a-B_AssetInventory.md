## §2 🔴 가장 중요한 결과 — CyberCity에는 **도로가 없다**

`road` / `pavement` / `sidewalk` / `asphalt` / `crossing` 이름의 메시가 **948개 중 0개**.
추측이 아니라 **Synty 자신의 데모 맵이 그 증거**다.

`/Game/PolygonCyberCity/Maps/Overview`(공식 데모, 의존 에셋 812개)가 지면에 쓰는 것:

### 3-1. 🟢 격자가 완벽히 맞는다 — 이게 결정적이다
```
건물 모듈  250
Scifi 도로 500  = 모듈 2개
1칸       1000  = 모듈 4개 = 도로 타일 2개
```
**500과 250 모두 1000(1칸)을 나눠떨어뜨린다.** 도로 키트를 그대로 우리 격자에 얹을 수 있다.
피벗 규약도 일정: 모든 타일이 `minX=0, minY=-500` → **모서리 피벗**, +X · -Y 방향으로 뻗는다.

### 3-2. 🔴 룩이 섞인다 — 사용자 육안 판정 필요
| 에셋 | 머티리얼 |
|---|---|
| CyberCity 전반 | `MI_PolygonCyberCity_01_A` |
| CyberCity 바닥 | `M_Floor_01` / `M_Floor_02` |
| **Scifi 도로** | **`M_PolygonScifi_Roads`** |
| **Scifi 인도/지면** | **`M_PolygonScifi_01_A`** |

**아틀라스가 다르다** = 색감이 섞일 수 있다. 같은 Synty 계열이라 스타일(로우폴리·셀)은 같지만
팔레트가 맞는지는 **눈으로 봐야** 안다 → S1에서 샘플 몇 장 깔고 사용자 확인.
드로우콜 영향은 **머티리얼 2개 추가** 수준(무시 가능).

### 3-3. ⚠️ 도로 타일을 쓸 때의 기술 함정 2개
1. **z-파이팅**: 도로 타일은 **두께 0**이고, 현재 바닥 평판 윗면도 정확히 **z=0**.
   그대로 얹으면 지글거린다 → **+1cm 띄우거나** 바닥 평판을 내려야 한다.
2. **콜리전 중복**: 도로 타일에도 콜리전이 1개씩 붙어 있다(두께 0인데도).
   바닥 평판이 이미 콜리전을 담당하므로 **타일은 `NoCollision`으로 까는 게 맞다**
   (플로우필드 쿼리 중복·베이크 비용 절감).

## §6-1 🔴 대각선 · 곡선 도로 = **불가** (2026-07-21 실측 확정, 재조사 금지)

사용자 질문("대각선이나 곡선 도로도 만들 수 있어?")에 대한 답 = **에셋이 없다.**

| 찾은 것 | 결과 |
|---|---|
| 도로 **대각선(45°) 타일** | 🔴 **0개** |
| 도로 **곡선 타일** | 🔴 **0개** |
| `SM_Env_Road_Lines_0N_Corner_01/02` (4종) | 곡선이 아니라 **90° 직각**. 정점 실측 = 차선 점선이 `X≈15.7~33.1`(Y방향)과 `Y≈-15.6~-33`(X방향) 두 줄로 직각으로 꺾인다 |
| `SM_Env_Sidewalk_Corner_*` | 인도 코너(연석이 아크로 돎). 도로 곡선용 아님 |

**건물 쪽 45° 조각도 사실상 못 쓴다**: `SM_Bld_Base_45_Wall_01` **벽 1종뿐** + `SM_Bld_Base_45_Floor_01`
(250×250, **삼각형 1개**) + `Walkway_45` + `Railing_45`. 벽이 한 종류라 대각선으로 세운 건물은
**창문 다양성이 0**이 되어 룩이 무너진다.

→ 대각선 거리를 굳이 하려면: 도로 타일을 45° 회전해 대각 격자로 깔고(기하학적으로는 맞물림),
접한 건물은 **계단식으로 두고 남는 삼각 자투리는 공터/광장/주차장으로** 채우는 방법뿐이다
(실제 도시의 대각선 대로변 삼각 부지 패턴). **사용자 결정 2026-07-21 = 하지 않음.**
