## §1 결정 (2026-07-23 사용자 확정)

- **아트 방향 = "변주 있는 Tron"** — 어두운 기하학적 건물 + **네온 엣지** + 간판 + **색 구역**.
  순수 풀-Tron(전부 검정+한 색)이 아니라, **Tron 화법을 통일 언어로 쓰되 구역 색·밝기·밀도를 변주**한다
  (레퍼런스 = 영화 *Tron: Legacy* 의 도시 장면. 그리드 공허가 아니라 색·엣지·간판이 있는 어두운 도시).

### 왜 이 방향인가 (제1원리)
1. **디테일 텍스처가 필요 없다** → "머티리얼 완성도"로 막혔던 지점을 우회. Tron은 어두운 표면 + 발광 엣지.
2. **싸다** → 발광+블룸은 저비용. **적 동시 200~300(캡 192) 예산에 유리**.
3. 🔴 **스웜 가독성 최고** → 1인칭에서 적 수백·투사체를 봐야 하는데, 어두운 배경 + 발광 실루엣 = 고대비.
   화면의 시각적 화려함은 **환경이 아니라 전투(적·스펠·폭발)** 가 낸다. 환경이 차분한 건 밋밋이 아니라 **무대**.
4. **안 맞던 머티리얼 문제가 사라진다** → 전부 "어두운 베이스 + 발광 엣지" 하나로 통일.
5. **블록형 모듈 지오메트리와 잘 맞는다**.

### 🔴 이건 기존 아트 결정에서의 피벗이다
- 기존 메모리 = **Synty + 애니 셀/툰**(무기 Synty Military / 캐릭터 애니 Blu / 적 VAT / UI Synty HUD).
  이 Tron 방향은 그와 **다른 목표**다. 충돌을 알고 진행한다.
- **환경 먼저 → 사용자 육안 게이트 → 게임 전체 전환 여부 결정**(§4 P2). 성급하게 전체 피벗 금지.
- CyberCity 팩은 원래 "지저분·빽빽 블레이드러너"용이라 팩의 디테일을 **안 쓰는 방향**이 된다(그게 핵심이자 이득).
  지오메트리(건물·도로 배치)는 그대로 재활용된다 — 헛일 아님.

## §3 목표 룩 정의 (구체적으로 — 밋밋 방지 체크리스트 겸)

| 요소 | 목표 |
|---|---|
| **건물 표면** | 어두운 베이스(거의 검정~짙은 남색) + **발광 엣지/패널 라인**. albedo 디테일 최소 |
| **색 구역** | 사분면/지구마다 지배 네온색(예: 광장=시안 / 동=마젠타 / 서=오렌지 / 북=그린 / 남=블루). **길찾기 + 다양성** |
| **밝기 계층** | 큰길 엣지 밝게, 뒷골목 흐리게 → 시선 유도. 전부 같은 밝기 금지 |
| **바닥** | 젖은/반사 바닥(roughness 낮게) → 네온이 반사돼 빛이 두 배 |
| **포그** | 거리 그라디언트 + 살짝 발광 틴트 → 깊이. 검은 공백 방지 |
| **블룸** | 강하게(발광이 번져야 Tron). Auto Exposure는 **고정**(4인 화면 출렁임 방지) |
| **간판** | 팩 네온 간판(`NeonSign`·`Billboard`·`Holo`)을 **발광 실루엣 악센트**로. 디테일 없이 생동감 |
| **스카이** | 어두운 하늘 + 지평선 발광. 밝은 주광 금지 |

🔴 **밋밋 방지 = ① 구역 색 ② 밝기 대비 ③ 밀도 변주(빽빽한 곳 vs 빈 곳) ④ 반사 ⑤ 실루엣(층수 이미 5~17층 다양).**
이 다섯을 게을리하면 밋밋해진다. 하나라도 빠지면 "검은 상자들"이 된다.

## §4 작업 순서

### P1 — 환경 프로토타입 (사용자 육안 게이트) ★여기부터
목표 = **밋밋한지 사는지 눈으로 판정할 수 있는 최소 룩**을 만든다. 커밋 전 되돌리기 쉽게.

1. **Tron 마스터 머티리얼** 제작(건물/도로용). 접근 옵션은 §5에서 **조사 후 택1**.
   - 파라미터: `BaseColor`(어두움), `EmissiveColor`(구역색), `EmissiveIntensity`, `EdgeWidth`, `Roughness`.
2. **구역별 색** = `MATERIAL_THEMES`(이미 있는 훅)를 Tron 머티리얼 인스턴스(구역색별)로 교체.
   - 건물의 사분면/지구에 따라 테마(=색) 배정하도록 `build_from_cubes`의 테마 선택을 위치기반으로.
3. **도로/바닥** = 어두운 반사 + 엣지 라인 머티리얼(격자/차선을 발광으로).
4. **SRS 재튜닝** = 노출 낮춤 / 블룸 강화 / 컬러그레이드(시안·틸) / 포그 그라디언트 / 리플렉션 roughness.
   - `BP_StylizedRenderingSystem`·`BP_SRS_Fog`·PostProcessVolume 파라미터를 실제로 읽고 조정.
   - ⚠️ M_Fog `Only on Custom Depth` 재발 주의(메모리).
5. **네온 간판 악센트** 몇 개 배치(발광 강조). 팩: `SM_Prop_NeonSign_*`·`Billboard_*`·`Holo_*`.
6. **사용자 육안 게이트** — 스크린샷 받아 ① 밋밋한가 ② 가독성 ③ 색 구역이 읽히나 판정.
   통과 못 하면 §3 다섯 레버 중 뭘 더 세게 할지 사용자와 정한다.

### P2 — 게임 전체 전환 (조건부, P1 게이트 통과 시에만)
🔴 **큰 결정. P1 없이 착수 금지.** 별도 계획 문서로 분리.
- 적 = **발광 실루엣**(어느 방향이든 가독성상 좋음 — 독립적으로도 남는 작업)
- 무기·캐릭터·UI·VFX 를 Tron 정합(기존 Synty/애니-셀 결정 재검토 필요)
- 이때 memory `synty-anime-cel-art-pivot` 를 이 결정으로 갱신/대체.

## §5 기술 접근 (P1 착수 시 **조사 후 결정**할 것들)

### 5-1. 엣지 글로우를 어떻게 낼 것인가 (핵심, 옵션 비교 후 택1 또는 조합)
| 옵션 | 방법 | 장/단 |
|---|---|---|
| **A. Fresnel rim** | 마스터 머티리얼에 Fresnel→Emissive | 쉬움·범용. 단 실루엣 테두리만 발광("림라이트"), 패널 라인 아님 |
| **B. 기존 emissive 재활용** | CyberCity 메시의 emissive 맵/네온 슬롯을 켜고 albedo 어둡게 | 팩 네온이 살아남. 단 메시마다 편차 |
| **C. SRS 아웃라인 재활용** | SRS 포스트프로세스 아웃라인을 **밝은 발광색 + 블룸**으로 재색 | 가장 "Tron"(실제 엣지 라인) + **기존 인프라 재활용**. 조사 필요 |
| **D. 그리드 머티리얼** | 어두운 베이스에 절차적 발광 그리드/패널 라인 | 싸고 통제 쉬움·매우 Tron. 바닥/벽에 특히 |
| **권장** | **C(SRS 아웃라인) + D(그리드) + B(간판 emissive)** 조합 | 실제 엣지 + 패널 라인 + 네온 악센트 |
→ **DESIGN-FIRST**: 새로 만들기 전에 SRS 아웃라인이 뭘로 구현됐는지(커스텀뎁스/스텐실/노멀 엣지) 먼저 조사.

### 5-2. 머티리얼 ↔ Bake ISM 상호작용 (이미 해결됨, 재활용)
- `bake_buildings`가 ISM 키를 `(메시, 콜리전, **머티리얼 시그니처**)`로 나눈다 → **색 구역별로 ISM이 갈린다**.
- 색(테마) 수가 늘면 드로우콜(ISM)이 는다. **구역 5색 정도가 다양성↔드로우콜 균형.**

### 5-3. 조사 방법
- SRS BP 파라미터: `execute_python_code`로 액터/컴포넌트 프로퍼티 덤프.
- 머티리얼 그래프: `export_material_graph`(신뢰 가능한 덤프, 메모리 `vibeue-material-graph-and-offline-render`).
- 인게임 룩은 MI 렌더 + 밉/TAA라 오프라인 렌더와 다름 → **PIE/뷰포트 실제 확인**(사용자 스크린샷).

## §6 제약 (제1원리)
1. **적 200~300 예산이 우선.** Tron 룩은 저비용이라 유리하지만, 블룸·리플렉션·볼류메트릭은 GPU 직격 —
   **적 200 스폰 상태에서 계측**(빈 맵 수치 무의미). 그림자 동적 광원 ≤4, 대부분 그림자 OFF.
2. **플로우필드는 머티리얼/PP만 바꾸니 무관**(콜리전·지오메트리 안 건드림). 룩 작업이 게임플레이를 안 깬다.
3. **가독성 최우선.** 어두워도 **적·투사체가 또렷이** 보여야 한다. 환경 발광이 적 발광을 덮으면 실패.
4. **4인 협동 기준.** Auto Exposure 고정(화면 출렁임 방지), 팀원 위치 식별 가능해야.
5. **에셋 경로 C++ 하드코딩 금지 / 서버권위·복제는 비주얼이라 무관.**
