# U3a 약점 부위 데미지 — 사용자 콘텐츠 / PIE 검증 가이드

> 코드는 `phase/p6-weakpoint-damage`에 머지 완료(빌드+스모크 통과). 이 문서는 **PIE 검증에 필요한 최소 콘텐츠**(약점 컴포넌트 배치 + 마커 틴트)를 따라하기 식으로 정리한다. 연출(전용 사운드/이펙트)은 보류.

## 0. 개요
약점 = 적/보스 BP에 디자이너가 배치하는 **`Weakpoint`(UFPSRWeakpointComponent, USphereComponent 파생)**. 명중 시 `DamageMultiplier` 배수. 코드가 위치값을 안 가지므로 **위치·크기·배수 모두 에디터에서 지정**.

## 1. 스웜 적 BP에 약점 배치 (검증용 최소 1개)
대상: 예) `BP_FPSREnemy`(또는 사용하는 스웜 적 BP).
1. BP 열기 → **Add Component** → 검색창에 **Weakpoint** → `Weakpoint` 추가.
2. 부착: **Capsule**(루트) 또는 **Mesh**에 드래그해 부착(상대 트랜스폼 고정 → 적이 움직이면 따라감).
3. 뷰포트에서 구를 **머리 위치로 이동**(예: 상대 위치 `Z ≈ +60`), **Sphere Radius**로 크기 조정(머리 크기).
4. 디테일 패널 **FPSR|Weakpoint → Damage Multiplier** = 예 `2.0`.
5. 컴파일·저장.
> **다중 약점(1~3개)**: 같은 방식으로 Weakpoint 컴포넌트를 더 추가(예: 머리 2.0 + 등 부패부위 1.5). 명중 시 **겹친 약점 중 가장 큰 배수 1개** 적용.

## 2. 약점 히트마커 틴트 (검증용 — 약점 식별)
대상: `WBP_HitMarker`(V3에서 만든 위젯).
1. `Setting Hit Marker`(또는 MarkerType 처리부) 그래프에서 **MarkerType** Switch에 **Weak** 케이스 추가.
   - `EFPSRHitMarkerType`에 `Weak`가 새로 노출됨(Hit/Crit/**Weak**/Kill).
2. Weak 케이스 → self `SetColorAndOpacity`에 **구분 색**(예: 노랑/주황). (Hit/Crit=흰, Kill=빨강과 구분)
3. 컴파일·저장.
> 그래프가 enum을 `Select`/`Switch on EFPSRHitMarkerType`로 처리 중이면 Weak 핀이 자동 추가됨 — 색만 지정.

## 3. PIE 검증 체크리스트
빌드 후 PIE(가능하면 2-client = 서버권위 확인):
- [ ] **약점 명중** = 본체보다 **추가 데미지**(배수만큼) + **약점 마커(구분 색)** 표시.
- [ ] **본체 명중** = 현행과 **완전 동일**(데미지·마커 Hit/Crit/Kill 변화 없음).
- [ ] **관통 무기(스나이퍼/관통 히트스캔)**로 약점+본체가 겹치는 적 사격 → **데미지 1회·관통 1회만** 소모(이중 적용 없음).
- [ ] **무기별**: 히트스캔(연사/샷건)·차지빔(워밍업 틱+페이오프 둘 다 약점 반영)·투사체 직격(스나이퍼/관통)·근접(칼) 모두 약점 동작.
- [ ] **AOE(바주카/유탄)** = 약점 **무반응**(폭발 데미지·마커 현행 동일).
- [ ] **약점 미배치 적** = 현행과 완전 동일(무회귀).
- [ ] **스웜 다수 스폰**(수십~수백) + 약점 배치 적 섞어 사격 → **프레임 회귀 없음**(§5 측정 대상).

## 4. 알려진 설계 결정 (버그 아님)
- **크릿 스택 = 곱연산 독립**: `Damage × DamageMult × 약점배수`, 크릿은 별도 `×CritMult`. 약점배수는 **마지막 곱**(크릿/프래그먼트 보너스 뒤).
- **차지빔 워밍업 틱도 약점 적용**(사용자 결정). 단 마커는 **페이오프만**(틱은 무음 — 마커 스팸 방지).
- **마커 우선순위**: Kill > Weak > Crit > Hit. 약점+크릿 동시 = Weak 표시(복합 마커는 후속 연출).
- **약점 위치 추종**: 상대 트랜스폼 부착이라 적 이동/회전에 자동 추종. 보스가 나중에 스켈레탈+애니가 되면 **본 소켓에 재부착**만으로 머리 추종(코드 변경 0, 후속 단순확장).

## 5. 성능 주의 (§5)
- 약점은 **데미지 쿼리에만** 응답(이동/지면/분리/플로우필드 무간섭, 전용 채널 ECC_FPSRWeakpoint).
- 풀 도먼트 적은 콜리전 자동 off → 활성 적만 약점 프리미티브 보유.
- **스웜 약점 권장 1~3개**. §5 본측정(P4-C 후 일괄)에 약점 query-result/dedup 비용 포함해 실측.
