## 0. 마인드셋 — 이건 "예쁜 도시"가 아니라 "게임플레이 셰이프 실험대"다
S1의 목적은 **CyberCity 매스로 스웜 서바이버 전투가 성립하는 공간을 만드는 것**이지, 아름다운 도시가 아니다. 파일럿이 검증하려는 건 세 가지뿐: **플로우필드가 건물을 장애물로 인식하는가(S1) → 셀 룩이 통하는가(S3) → 200-300 스웜이 프레임예산 안인가(S4)**. 따라서 **속도 > 미려함**. 큰 매스로 빠르게 도시 실루엣을 잡고, 디테일은 나중. 화이트박스는 언제든 부수고 다시 짤 수 있다.

## 1. 게임 제1원리로부터의 설계 고려사항 (WHY — 이게 핵심)

이 게임은 **1인칭 × 뱀파이어서바이벌 × 4인 협동**이다. 도시 형태가 아래 5가지를 좌우한다.

### 1-1. 스웜 스케일 & 1인칭 가독성 (`Performance §5`)
동시 적 **~200-300**(캡 500)이 몰려온다. 1인칭이라 시야가 좁고, **화면에 보이는 적이 너무 많으면 "에임이 의미 없는 난사"**가 된다. §5 설계 목표: 화면 내 보이는 적 **P50 ≤ 40 / P90 ≤ 70**, 15m 내 즉시위협 **P90 ≤ 25**.
- **함의:** 200m 직선 대로 하나에 스웜 전체가 보이게 하면 가독성 붕괴. **건물로 시야를 끊어** 한 번에 보이는 적 수를 제어하라. 코너·블록이 스웜을 "쪼개서" 밀려오게 만든다.
- **동시에** 너무 좁은 미로면 1인칭에서 답답 → 열린 플라자와 좁은 골목을 **섞어라**.

### 1-2. Back-to-back 협동 USP (`Concept §1-C-1~7`)
이 게임의 차별점 = **1인칭이라 각자 등 뒤가 안 보여서 서로 엄호**해야 한다. 도시가 이 순간을 만들어야 한다.
- **360° 위협 공간**(중앙 플라자/교차로)을 둬라 — 적이 사방에서 오면 4인이 자연히 등을 맞대고 각 방향을 맡는다. 이게 USP가 빛나는 무대다.
- **엄호 섬(cover island)**: 플라자 중앙·교차로에 낮은 커버(허리 높이 벽/프롭)를 흩어두면 "내 사각 좀 봐줘"가 발생한다. 커버가 전혀 없으면 그냥 뺑뺑이, 너무 많으면 스웜이 못 온다.

### 1-3. 플로우필드 계약 (`Performance §5-2` / 검증된 툴 규칙)
적 길찾기 = **Flow-Field**(고정맵 사전계산). 건물이 장애물로 인식되려면 **반드시 WorldStatic BlockAll**이어야 한다.
- **S0 블록아웃 툴 '뷰포트 배치'로 놓은 StaticMesh = 자동 WorldStatic BlockAll** → OK.
- ⚠️ **CyberCity의 액터 BP(Blueprints 폴더 63개)는 as-is 스폰**(자동 변환 없음) + **검증기가 콜리전 검사 안 함** → BP 건물은 플로우필드가 못 볼 수 있다. **건물 매스는 `Meshes/Buildings`의 StaticMesh로 배치**하라(BP 지양).
- **통로 폭**: 플로우필드 셀은 기본 **200cm**. 좁은 골목은 셀 경계에서 **과차단**되는 트레이드오프가 있다(§5-2 C1). **길·골목은 최소 4-6m(2-3셀) 폭**을 줘라. 1셀(2m) 폭 틈은 적이 못 지나갈 수 있다.

### 1-4. 스폰 로직 (`§5-2` / `enemy-spawnpoint-z-floor-offset`)
스폰 디렉터는 **디자이너 배치 스폰포인트**에서 **전 플레이어 비가시(FOV 밖) + 거리 게이트**로 가중랜덤 스폰한다.
- **페리미터에 분산 배치** → 스웜이 사방에서 밀려와 중앙에서 수렴(back-to-back 무대 성립). 한쪽에만 두면 한 방향 난사가 된다.
- **건물 뒤/코너 너머**에 둬라 → 적이 시야 밖에서 스트리밍돼 "코너를 돌아 나타남"(팝인 방지, 가독성 유지).
- **Z = 바닥 + 100cm**(캡슐 반높이 90; 바닥에 딱 붙이면 발이 묻혀 낙하). `MinPlayerDistance` 1500~3000(15~30m).

### 1-5. 수직성 — 유계 2층 (`§5-2 U7`)
플로우필드는 **겹친 2층(메자닌/2층 덱)**까지 지원(계단·램프로 2층 플레이어 추격). 3층+·동일 XY 두 경사면 교차는 미지원.
- (선택) 플라자 위 **раised walkway 1개**나 2층 덱을 storey급(~1층 높이)으로 얹으면 "위층으로 도망→적이 계단으로 추격" 다이내믹을 파일럿에서 확인 가능. 필수는 아님.

### 1-6. 통일 모티프 — 4방향 거대 문 (`Concept §1-C-9`)
맵마다 테마가 달라도 **4방향 끝의 거대한 문**이 심(sim) 간 전환 포털이자 통일 요소다. 화이트박스에도 **동서남북 가장자리에 큰 문틀 4개**(teal `MI_Dev_Door`)를 두면 방향 감각·스폰 앵커·픽션 정합을 동시에 잡는다.

## 2. 치수·수치 가드레일 (구체 수치)

| 항목 | 값 | 근거 |
|---|---|---|
| **플레이 영역 크기** | **한 변 ~180-240m 권장, 최대 ~380m** | 셀예산 40000 ÷ (200cm 셀) = 200×200셀 = **400m×400m가 상한**. 컴팩트할수록 "적이 항상 근처" 서바이버 감 + 셀 여유 |
| 플로우필드 셀 | 기본 200cm (볼륨 `CellSizeOverride`로 조절) | 축당 ≤256셀 / 총 ≤40000셀 = **하드 에러 라인** |
| 길·골목 폭 | **≥ 4-6m (2-3셀)** | 좁으면 과차단(적 통과 불가) |
| 중앙 플라자/아레나 | ~40-60m 열린 공간 | 360° 위협 + back-to-back 무대 |
| 건물 높이 | 다양하게 (저층~고층 매스) | 시야 차단·실루엣, 다만 스웜은 지상 추격 |
| 스폰포인트 Z | **바닥 + 100cm** | 캡슐 반높이 90 |
| 스폰 `MinPlayerDistance` | 1500~3000 | 시야 밖 스트리밍 |
| 볼륨 `MapId`/`bUnifiedExtent` | **미설정** | 단일 throwaway 맵 |

> **셀예산 계산 팁:** 볼륨 박스 XY를 `200cm × 200cm`로 나눈 값이 셀 수. 예) 240m×240m = 120×120 = 14,400셀(여유). 400m×400m = 40,000셀(딱 상한). 넘으면 '레벨 검증'이 **에러** → 볼륨을 줄이거나 볼륨의 `CellSizeOverride`를 250~300으로 키운다(툴바 '격자'가 아니라 **볼륨 프로퍼티**임에 주의).

## 3. 권장 매크로 레이아웃 — "컴팩트 링 시티"

빠르게 좋은 스웜 전투 공간을 얻는 검증된 형태:

```
        [北 문]
   ┌───────────────┐
   │ 블록  스폰○   │      ○ = 스폰포인트(건물 뒤, 페리미터)
[西 │   ┌─────┐    │ 東    ▦ = 중앙 플라자(360° 아레나 / 미션·보스 홀 후보)
 문]│스폰○│ ▦▦▦ │블록 │ 문   ─│ = 스트리트(≥6m, 방사형 또는 그리드)
   │   └─────┘ ○스폰│      □ = 건물 매스(오렌지 WorldStatic)
   │ 블록    블록   │      · = 커버 섬(플라자·교차로 낮은 커버)
   └───────────────┘
        [南 문]
```

- **중앙 플라자**(~50m) = USP 무대. 여기에 미션 홀존/보스 아레나를 얹기 좋다. 안에 커버 섬 2-3개.
- **방사형(또는 격자) 스트리트** 4개가 플라자→4방향 문으로. 폭 ≥6m. 스웜의 방향성 위협 통로.
- **건물 블록**이 스트리트 사이를 채움 — 높이 섞어 시야 차단·코너 생성.
- **4방향 문**(teal) = 포털+스폰 앵커. 문 안쪽/건물 뒤에 스폰포인트 분산.
- (선택) 플라자 한쪽에 **2층 덱 + 계단** 1개(U7 추격 검증).

> **대안(더 단순):** 균일 격자 도시 — 6×6 블록에 격자 스트리트. 만들기 쉽고 플로우필드 예측 쉬움. 다만 360° 아레나 순간이 약하니 중앙 1블록을 비워 플라자로.

## 4. 개발자 머티리얼 사용법 (오렌지 박스톤)

`Scripts/gen_blockout_materials.py` 실행 시 `/Game/_SyntyPilot/DevBlockout/`에 생성됨. **월드 정렬 그리드**라 어떤 메시에 붙여도 실제 스케일 격자선이 나온다(거리 가늠용).

| 머티리얼 | 색 | 용도 |
|---|---|---|
| `MI_Dev_Building` | 오렌지 | 건물 = 솔리드 장애물(스웜 못 통과) |
| `MI_Dev_Wall` | 진한 앰버 | 내벽·낮은 커버 |
| `MI_Dev_Road` | 차콜(거의 흑) | 도로·차선(보행 가능, 스웜 흐름) |
| `MI_Dev_Ground` | 중간 회색 | 플라자·지면 채움 |
| `MI_Dev_Zone_Spawn` | 빨강(발광) | 스폰 구역 마커 |
| `MI_Dev_Zone_Mission` | 파랑(발광) | 미션·홀드 구역 |
| `MI_Dev_Zone_Boss` | 보라(발광) | 보스 아레나 |
| `MI_Dev_Door` | 청록(발광) | 4방향 포털 문 |

**색 규칙 = 게임플레이 니모닉:** 오렌지=막힘(장애물), 어두운 회색=뚫림(보행로). 존 발광색은 지면 패치/데칼로 깔아 스폰·미션·보스 위치를 한눈에.


## 5. 빌드 순서 (검증된 S0 툴 워크플로 반영)

1. **맵 복제** — `L_SyntyPilot`(또는 현 테스트 맵) 복제 → `Content/_SyntyPilot/L_SyntyPilot_CyberCity`.
2. **머티리얼 생성** — §4 스크립트 실행(1회).
3. **WorldStatic 지면 먼저** — 큰 Cube(또는 바닥 메시)를 Z=0에 깔고 `MI_Dev_Ground` 적용, 콜리전 WorldStatic BlockAll 확인. *(⚠️ 뷰포트 배치 고스트가 WorldStatic 바닥에 스냅하므로 지면이 먼저 있어야 건물이 깔끔히 붙는다.)*
4. **팔레트 확인** — `Project Settings > FPSR > FPSR Blockout > Palette Folders`에 `/Game/PolygonCyberCity/Meshes` 확인(기본값). 좌측 폴더 리스트에서 **Buildings / Base / Environment**를 클릭해 쓴다(Characters/Weapons/Vehicles는 무시).
5. **건물 매스 배치** — `Tools>FPSR>블록아웃 툴` → 카드 선택 → **'뷰포트 배치'** → 커서로 바닥에 붙여 좌클릭 연속 배치(자동 WorldStatic). §3 링시티 실루엣 구성. 툴바 '격자(cm)'로 스냅 조절. `MI_Dev_Building`/`MI_Dev_Wall` 적용. *(연속 배치엔 겹침 가드 없음 — 자유 배치 정상.)*
6. **스트리트·플라자 비우기** — 건물 사이로 ≥6m 길, 중앙 ~50m 플라자 확보. 커버 섬(낮은 프롭) 배치.
7. **FlowFieldBoundsVolume 1개** — `AFPSRFlowFieldBoundsVolume` 배치, BoundsBox를 플레이 영역 전체로. **박스 중심에 콜리전 액터 두지 말 것**(오앵커, `flowfield-volume-center-collision-floortrace`). `MapId`/`bUnifiedExtent` 미설정.
8. **스폰포인트** — `AFPSREnemySpawnPoint`를 페리미터·건물 뒤에 분산, **바닥+100**. `MinPlayerDistance` 1500~3000.
9. **4방향 문** — 가장자리 4곳에 큰 문틀(`MI_Dev_Door`).
10. **레벨 검증** — 블록아웃 툴 **'레벨 검증'** → `FPSRBlockout` Message Log 6종 가드레일 통과(§6).
11. **PIE 스웜 테스트** — ⚠️ **World Settings > GameMode Override = FPSR 게임모드**(적=`BP_EnemyMeleeBase`) 지정 후 → `FPSR.EnemyTarget 200` → 적이 건물을 **돌아서** 추격하는지(플로우필드 장애물 인식) 확인. `FPSR.RunDebug 1`.

## 7. 흔한 함정 (코드 검증에서 확인)
- **⛔ MapId 태그 = 단일맵 스폰 완전차단 (WhiteBox 복제 최대 함정, 2026-07-14 실측).** L_U_Whitebox는 3슬롯 멀티맵 리그다 — 복제하면 볼륨·스폰포인트가 슬롯 태그(`Map.Center` 등)를 물려받는다. 단일맵 PIE엔 `bUnifiedExtent` 볼륨이 없어 플로우필드가 **디그너릿 단일 그리드**를 만들고(`SlotBounds` 빈 채) 플레이어가 **Default(빈 태그)로 해석**된다. 스폰 디렉터는 `Point.MapId != TargetMapId`로 필터하므로 `Map.Center` 스폰은 Default와 안 맞아 **후보 0 → 적 한 마리도 안 나옴**(링폴백·FOV게이트 제거됨). 진단 = 로그 `Warning: [FlowField] ... without a bUnifiedExtent volume — built a single degenerate grid` + 그리드가 `(origin-centered fallback)`. **수정 = 볼륨 + 전 스폰포인트 `map_id`를 Default(빈 태그)로 언태그** → 그리드가 `(bounds volume)`로 정확히 사이징되고 스폰 매칭됨. **다음 맵은 WhiteBox가 아니라 단일맵 베이스(L_Sandbox류)에서 복제**할 것. (메모리 `mapid-tag-single-map-spawn-block`)
- **툴바 '격자'(스냅) ≠ 플로우필드 셀 크기.** 셀 예산은 **볼륨 `CellSizeOverride`(기본 200)**로만 바뀐다.
- **뷰포트 연속 배치엔 겹침 가드 없음**(버튼 '선택 배치'에만 있음) — 도시 쌓기엔 정상.
- **액터 BP 건물은 플로우필드가 못 볼 수 있음** → StaticMesh 우선.
- **지면(WorldStatic)을 건물보다 먼저** 깔아야 고스트 바닥 스냅이 깔끔.
- **PIE 스폰은 GameMode Override 필수** — 안 하면 적 0.
- 편집 후 같은 세션 PIE = World Leak 위험 → **편집 후 에디터 재시작**(`vibeue-buildgraph-pie-worldleak`).
