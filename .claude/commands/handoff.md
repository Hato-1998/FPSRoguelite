---

컨텍스트가 차서 새 세션으로 인계한다. 아래를 순서대로 수행하라.

1. **검증 확인 (멈춤 게이트)**: 현재 브랜치에 미커밋 *코드*(`Source/`·`Scripts/`)가 있으면, **빌드+스모크가 통과한 상태인지 확인**한다. 미검증·실패 상태면 커밋하지 말고 사용자에게 알린 뒤 멈춘다(프로젝트 규칙: 검증 없이 커밋 금지).

2. **PROGRESS.md 핸드오프 갱신**: `git branch --show-current` 기준으로 상단 핸드오프 영역에 간결히 기록한다 —
   ① 이 브랜치에서 한 일(완료 커밋 1줄 요약) ② **다음에 할 코드 작업(구체적, 파일/함수 단위)** ③ 블로커/주의 ④ 미커밋 콘텐츠 목록(`git status --short`의 `Content/` 항목 = 사용자 작업으로 남김).

3. **코드/문서만 커밋**: `Source/`·`Scripts/`·`*.md`(PROGRESS/Game.MD 등)만 `git add` 후 커밋한다. **`Content/`(에셋·맵·BP)는 절대 커밋하지 말 것** — 사용자 콘텐츠 작업으로 남긴다. 커밋 메시지는 프로젝트 컨벤션(끝에 `Co-Authored-By: Claude ...`).

4. **재개 프롬프트 출력**: 새 세션에 그대로 붙여넣을 한 줄을 출력한다. 형식 예:
   `phase/<브랜치> 코드작업 이어서 진행 — PROGRESS.md 핸드오프 영역 + Game.MD 먼저 읽고, <다음 작업>부터. 콘텐츠/PIE 없이 코드만, 검증=빌드+스모크+Codex.`

5. **푸시 여부 질문**: 다른 기기(핸드폰 등)에서 이어가려면 push 필요함을 알리고, 원하면 `git push origin <브랜치>` 수행 여부를 묻는다.

인자 `$ARGUMENTS` 가 있으면 인계 메모에 반영한다(예: 특정 다음 작업 지정).
