게임 컨설턴트 토론을 수행한다. **반드시 먼저 [`Docs/ConsultLoop.md`](Docs/ConsultLoop.md) 전체를 읽고 그 프로토콜대로** 진행하라. (그 문서가 SSOT — 아래는 요약일 뿐.)

토론 주제: **$ARGUMENTS**
- 인자가 비어 있으면, 먼저 무엇을 토론할지 사용자에게 물어본 뒤 진행한다.

수행 순서 (ConsultLoop §3):
1. **R0 범위설정** — `Game.md`·`PROGRESS.md` + 주제 관련 `Docs/SSOT/*.md`와 관련 소스/콘텐츠를 읽어 **사실 기반**을 잡고, 핵심 쟁점 2~4개로 의제를 프레이밍한다(추측 금지).
2. **R1~ 토론 루프** — 너는 **백엔드/시스템 렌즈**다. 라운드마다: ① 백엔드 입장 작성 → ② 프롬프트 파일에 (역할 프라이머 + 의제/합의/쟁점 요약 + 백엔드 입장 + 구체 질문 2~4개)를 적고 → ③ `Scripts/consult-codex.ps1 -PromptFile <file> -Title <슬러그>` 로 **클라이언트 렌즈(Codex)** 응답을 받는다 → ④ 판독·반박·조정 후 다음 라운드. **새 실질 논점이 안 나올 때까지** 반복(시간 제한 없음, 하드 캡 없음).
3. **수렴 정리** — (a)합의 / (b)미해결 쟁점 / (c)사용자 결정 필요 로 분류.
4. **산출** — ConsultLoop §5 템플릿으로 `Docs/Review/<yyyyMMdd>-<슬러그>.md` 저장 + 사용자에겐 **압축 요약**(합의/쟁점/결정필요/액션)을 채팅 보고.

가드레일 (ConsultLoop §6): **자문 전용 — 코드/에셋 변경 금지.** 채택 설계는 별도 단계에서 해당 `Docs/SSOT/` 파일을 먼저 갱신. 두 렌즈 모두 장르 정체성(`Game.md` §1, **Hero Shooter 아님**)·제1원리에 구속. Codex 미설치/미인증이면 `npm install -g @openai/codex` + `codex login` 안내.
